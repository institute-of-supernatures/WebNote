1.1-事件的另一种添加方式

  • 1.注册事件第二种方式

    • 对象.addEventListener(参数1,参数2,参数3)
      • 参数1：事件名(字符串)，不要加on 例如：click 、 mouseover 、mouseout
      • 参数2：事件处理程序（函数名），当事件触发后哪个函数来处理
      • 参数3：是一个bool类型，可以不传，默认为fasle（代表冒泡）跟冒泡和捕获有关
    • IE8及之前使用:对象.attachEvent(参数1，参数2)
    • 可以给对象添加多个同名事件，如果有同名事件不会覆盖，而是会依次执行

  • 2.移除事件

    • 元素名.removeEventListener(参数1，参数2，参数3);
    • 元素名.事件名 = null
    • 元素名.detachEvent(参数1，参数2)
      • IE8及之前

1.2-事件冒泡与捕获

  • 1.事件冒泡

    • 1.事件冒泡：如果一个元素的事件被触发，那么他的所有父级元素的同名事件也会被依次触发
      • 元素->父元素->body->html->window
      • 事件冒泡一直存在，只不过以前我们没有给父级元素加同名事件
    • 2.事件冒泡好处：如果想给父元素的多个子元素添加事件，我们可以只需要给父元素添加事件即可，然后 通过获取事件源(e.target)就可以得知是哪一个子元素触发了这个事件
    • 3.事件冒泡影响：事件冒泡会导致需求冲突
    • 4.阻止事件冒泡
      • 事件对象.stopPropagation()
      • 事件对象.cancelBubble = true
        • IE8及之前的方式
      • 注意：如果想要阻止事件冒泡，一定要在触发事件的函数中接收事件对象

  • 2.事件捕获

    • 1.事件捕获：从最顶级的父元素一级一级往下找子元素触发同名事件，直到触发事件的元素为止
      • 事件捕获与事件冒泡触发事件的顺序完全相反
    • 2.事件捕获，只能通过addEventListener并且参数写true才是事件捕获
      • 其他方式添加都是冒泡，（不是通过addEventListener添加、addEventListener参数为false）
    • 3.事件对象.stopPropagation() 除了可以阻止冒泡还可以阻止捕获
    • 4.IE8及以前没有捕获！

1.3-事件三个阶段及事件类型

  • 1.事件三个阶段

    • 1-捕获阶段
    • 2-目标阶段
    • 3-事件冒泡
    • 事件对象.eventPhase 可以获得触发这个事件时，到底是哪个阶段
    • 这三个阶段代表的是事件执行的顺序，先执行捕获型事件，然后是目标事件，然后是冒泡型事件

  • 2.事件类型

    • 事件对象.type 能获取到触发事件的事件名
      • 去掉前面的on，例如onclick获取到的是click

1.4-键盘事件

  • 键盘事件
    • onkeyup：键盘弹起触发
    • onkeydown:键盘按下触发
    • onkeypress：键盘按下触发（用的比较多）
    • onkeydown和onkeypress的区别：
      • 1.onkeypress可以过滤掉特殊的功能键例如删除、F1-F12,shift,alt键等等，onkeydown不会过滤
      • 2.onkeypress可以区分大小写，但是onkeydown永远都是大写（不管大小写状态）
    • 如何获取你到底按的是哪个键？
      • 事件对象.keyCode
        • 存在浏览器兼容
        • 获取到的是键盘对应字符的ascii码
        • ascii码转字符：String.fromCharCode(code)
      • 解决浏览器兼容：var code = e.keyCode || e.charCode || e.which;
