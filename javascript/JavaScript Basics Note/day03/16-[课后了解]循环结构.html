循环结构

  • 本内容与课堂内容一致，只是摘抄自其他技术网站描述，语言略微生涩，仅供参考

  • 循环结构：重复的做一件事

    • 例如：用代码实现从1加到100的总和，如果用之前的代码实现，我们需要写一百次加法，使用循环之后我们只需要让代码循环一百次添加即可

  • 循环结构三要素：循环变量，循环条件，循环变量自增

    • 循环变量：记录循环当前次数
    • 循环条件：决定循环是继续还是停止
    • 循环变量自增：控制循环的次数，否则将会导致死循环

  • js的循环语句有三种：

  • 1.for循环语法如下


for循环的语法：
            for( 单次表达式1;条件表达式2;末尾循环体3 ){

                  中间循环体;
            }
       执行过程：
                1.首先执行表达式1,该表达式不参与循环，只会执行一次
                2.执行表达式2，这是for循环的正式开端
                    * 如果为true，执行中间循环体代码
                    * 如果为false，循环结束
                3.执行末尾循环体
                4.重复步骤2

  • for循环详细讲解

    • 1.for循环小括号里第一个“;”号前为一个为不参与循环的单次表达式，其可作为某一变量的初始化赋值语句, 用来给循环控制变量赋初值； 也可用来计算其它与for循环无关但先于循环部分处理的一个表达式。
    • 2.俩“;”号之间的条件表达式是一个关系表达式, 其为循环的正式开端，当条件表达式成立时执行中间循环体。
    • 3.执行的中间循环体可以为一个语句，也可以为多个语句，当中间循环体只有一个语句时，其大括号{}可以省略，执行完中间循环体后接着执行末尾循环体
    • 4.执行末尾循环体后将再次进行条件判断，若条件还成立，则继续重复上述循环，当条件不成立时则跳出当下for循环

  • 2.while循环


 while语法：
                while(循环条件){

                    循环体;
                }


        循环体：要重复执行的代码

        语义：
             1.先判断条件是否为true，如果为true，跳转到2，如果为false，跳转到3
             2.执行循环体，执行完毕，又跳回1
             3.跳出循环，循环结束

  • 3.do-while循环

  • 与while循环最大的区别是：do-while循环无论循环条件是否成立都会首先执行一次循环体


do-while语法：
        do{

            循环体;

        }while( 循环条件 );

    语义；
         1.执行循环体，执行完毕，跳转到2
         2.判断条件是否为true，如果为true，则跳回1，如果为fasle，则跳到3
         3.结束循环，跳出循环

    do-while和while实现的循环其实是一样的，一个不同点：

    do-while的循环体至少会被执行1次，而while可能一次都不执行

1.1-for循环总结

  • for循环的标准语法是for(;;){}

    • 小括号的两个分号缺一不可
    • 大括号也可以省略，但是不规范
    • 也就是说：for循环的所有表达式都可以省略，但这样是一个死循环，没人会做这种lowB的事情

  • for循环的规范语句应该是for(声明循环变量;循环条件判断;循环变量自增/自减){循环体}

    • 循环变量也可以在for循环外部声明，只是会多一行代码
    • 循环条件判断常用于决定循环的次数
    • 循环变量的自增与自减也可以写在循环体中（不规范）

1.2-while循环总结

  • while循环的语法看起来比for循环稍微简洁一点，但实际上需要做的事情还是一样
    • 循环变量
    • 循环条件
    • 循环变量自增或自减

while(条件){

    循环体;
}

1.3-do-while循环总结

  • 从语法结构的区别可以看出，do-while循环其实只是把循环体提前
    • 这样做的结果就是先执行一次循环体，然后在判断循环条件

do{
    {

        循环体;
    }
}while(条件)

1.4-三种循环语句的使用场景选择

  • *在原则上，三种循环语句可以完全转换，使用哪一种都可以实现功能

  • 1.优先使用for循环的场景

    • （1）循环次数确定
    • （2）用于代码计算
      • 例如计算1-100的和

  • 2.优先使用while循环的场景

    • （1）循环次数不确定
    • （2）用于完成程序某个功能的业务逻辑
      • 例如用户登陆：只有账号密码正确的情况下才会成功否则一直提示登陆
        • 循环次数不确定，不需要计算，只是为了完成某个功能的业务逻辑

  • 3.优先使用do-while循环的场景

    • do-while的应用场景基本与while循环一致
    • 如果不确定循环的次数，但是循环至少要执行一次，优先使用do-while循环
      • while循环会先判断循环条件，如果循环条件不成立，可能循环体代码一次都不会执行
